Prerequisites

Before installing Neo-RS, ensure your system meets the following requirements:

üíª System Requirements

  • CPU: 2+ cores (4+ recommended)
  • RAM: 4GB minimum (8GB+ recommended)
  • Storage: 50GB+ free space (SSD recommended)
  • Network: Stable internet connection

üñ•Ô∏è Supported Platforms

  • Linux (Ubuntu 20.04+, RHEL 8+, Debian 11+)
  • macOS (10.15+)
  • Windows (10/11, Server 2019+)
  • Docker containers
üí° Tip: For production deployments, we recommend using Linux with an SSD for optimal performance.

Installation

Choose the installation method that best fits your needs:

Binary Download (Recommended)

The fastest way to get started is downloading a pre-built binary:

Linux x86_64 
# Download and install Neo-RS
curl -L https://github.com/r3e-network/neo-rs/releases/latest/download/neo-rs-linux-x86_64.tar.gz | tar xz
sudo mv neo-rs /usr/local/bin/
neo-rs --version
macOS 
# Download and install Neo-RS for macOS
curl -L https://github.com/r3e-network/neo-rs/releases/latest/download/neo-rs-macos-x86_64.tar.gz | tar xz
sudo mv neo-rs /usr/local/bin/
neo-rs --version
Windows PowerShell 
# Download Neo-RS for Windows
Invoke-WebRequest -Uri "https://github.com/r3e-network/neo-rs/releases/latest/download/neo-rs-windows-x86_64.zip" -OutFile "neo-rs.zip"
Expand-Archive -Path "neo-rs.zip" -DestinationPath "C:\neo-rs"
# Add C:\neo-rs to your PATH environment variable
neo-rs.exe --version

Build from Source

Build Neo-RS from source for the latest features or custom configurations:

‚ö†Ô∏è Note: Building from source requires Rust 1.70+ and system dependencies.

Install Rust

Install Rust toolchain 
# Install Rust
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
source $HOME/.cargo/env
rustc --version

Install System Dependencies

Ubuntu/Debian 
sudo apt-get update
sudo apt-get install -y \
    build-essential \
    pkg-config \
    libssl-dev \
    librocksdb-dev \
    clang
macOS 
# Install Homebrew if not already installed
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install dependencies
brew install rocksdb

Build Neo-RS

Clone and build 
# Clone the repository
git clone https://github.com/r3e-network/neo-rs.git
cd neo-rs

# Build with optimizations
cargo build --release --all-features

# Install to system
sudo cp target/release/neo-rs /usr/local/bin/
neo-rs --version

Package Manager

Install Neo-RS using your system's package manager:

Homebrew (macOS/Linux) 
# Add Neo tap
brew tap neo-project/neo

# Install Neo-RS
brew install neo-rs
neo-rs --version
Cargo (Rust) 
# Install from crates.io
cargo install neo-rs

# Make sure ~/.cargo/bin is in your PATH
neo-rs --version
üí° Note: Package manager installations may not have the latest version. Check the releases page for the most recent version.

Docker

Run Neo-RS in a Docker container for easy deployment:

Quick start 
# Run Neo-RS on TestNet
docker run -d \
  --name neo-rs \
  -p 10332:10332 \
  -p 10333:10333 \
  -v neo_data:/data \
  neo/neo-rs:latest --testnet

# Check logs
docker logs -f neo-rs
With custom configuration 
# Create configuration file
cat > neo-config.toml << EOF
[network]
network = "testnet"
p2p_port = 10333
rpc_port = 10332

[storage]
data_dir = "/data"
cache_size = "1GB"

[rpc]
enabled = true
bind_address = "0.0.0.0:10332"
EOF

# Run with custom config
docker run -d \
  --name neo-rs \
  -p 10332:10332 \
  -p 10333:10333 \
  -v ./neo-config.toml:/etc/neo/neo-config.toml:ro \
  -v neo_data:/data \
  neo/neo-rs:latest

Basic Configuration

Create a basic configuration file to customize your Neo-RS node:

neo-config.toml 
# Basic Neo-RS configuration
[network]
network = "testnet"          # "mainnet", "testnet", or "privatenet"
p2p_port = 10333            # P2P network port
rpc_port = 10332            # JSON-RPC port
max_peers = 100             # Maximum number of peers

[storage]
data_dir = "./data"         # Data directory path
cache_size = "1GB"          # Memory cache size
compression = "lz4"         # Compression algorithm

[rpc]
enabled = true              # Enable JSON-RPC server
bind_address = "0.0.0.0:10332"  # RPC bind address
max_connections = 1000      # Maximum RPC connections
cors_origins = ["*"]        # CORS allowed origins

[consensus]
enabled = false             # Enable for validator nodes only
# wallet_path = "./wallet.json"  # Wallet file for consensus

[logging]
level = "info"              # Log level: error, warn, info, debug, trace
format = "pretty"           # Log format: pretty, json

Configuration Tips

üîí Security

  • Bind RPC to localhost for local access only
  • Use firewall rules to restrict access
  • Never expose wallet files publicly

‚ö° Performance

  • Increase cache_size for better performance
  • Use SSD storage for data directory
  • Adjust max_peers based on bandwidth

üîß Development

  • Use "testnet" for development
  • Enable debug logging for troubleshooting
  • Set lower cache sizes for testing

Running Your Node

Start your Neo-RS node with different network configurations:

TestNet (Recommended for beginners) 
# Run on TestNet
neo-rs --testnet

# With custom data directory
neo-rs --testnet --data-dir ./neo-testnet-data

# With custom configuration
neo-rs --config neo-config.toml
MainNet (Production) 
# Run on MainNet
neo-rs --mainnet

# With specific ports
neo-rs --mainnet --rpc-port 10332 --p2p-port 10333

# Run in background (systemd recommended for production)
nohup neo-rs --mainnet > neo.log 2>&1 &
Private Network 
# Run private network
neo-rs --privatenet

# With specific genesis block
neo-rs --privatenet --genesis ./genesis.json
‚úÖ Success! Your Neo-RS node will start synchronizing with the network. This may take several hours for the initial sync.

Verification

Verify that your node is running correctly:

Check Node Status

RPC Health Check 
# Check if RPC server is responding
curl -X POST http://localhost:10332 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "getblockcount",
    "params": [],
    "id": 1
  }'

# Expected response:
# {"jsonrpc":"2.0","id":1,"result":5847291}

Monitor Synchronization

Check sync status 
# Get current block height
curl -X POST http://localhost:10332 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "getblockcount",
    "params": [],
    "id": 1
  }'

# Get peer information
curl -X POST http://localhost:10332 \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "method": "getpeers",
    "params": [],
    "id": 1
  }'

View Logs

Monitor node logs 
# If running in foreground, logs appear in terminal
# If running with nohup
tail -f neo.log

# If running with systemd
journalctl -u neo-rs -f

# If running with Docker
docker logs -f neo-rs
