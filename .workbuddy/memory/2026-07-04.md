  neo-blockchain implement neo-engine::PipelineStage. Phase 1 (ValidateStage)

  is high effort (unwind command loop coupling).

  9 crates use CoreError directly. Rule: domain-specific = own error,
  validation/codec = CoreError. All domain errors have From<DomainError> for CoreError.
- **ADR-012**: MPT layering documented — neo-crypto::mpt_trie (data structure)
  + neo-state-service::mpt_store (durable store) are layered, NOT duplicated.
  neo-crypto::mpt_trie also used directly by neo-rpc (proof verification).
- **ADR-013**: doc(html_root_url) version management — fixed and documented.
- **ADR-014**: RuntimeVersion type — proposed for long-term, deferred until
  multi-version support needed.
- **ADR-015**: Native contract registry extensibility — proposed for long-term,
  deferred until runtime-loadable contracts needed.

### Key Findings (No Action Needed)
- **MPT is NOT duplication**: Layered design — neo-crypto owns data structure,
  neo-state-service owns durable store, neo-rpc uses data structure directly.
- **neo-vm/neo-vm-rs boundary is clean**: neo-vm-rs = pure semantics (no_std),
  neo-vm = stateful host (RC, execution context). Documented boundary in lib.rs.
- **neo-storage/neo-state-service boundary is clean**: storage = generic KV store,
  state-service = MPT state root layer on top.
- **neo-execution/neo-native-contracts/neo-engine division is clean**:
  execution = engine + trait, native-contracts = 11 implementations,
  engine = pipeline abstraction (scaffolded).
