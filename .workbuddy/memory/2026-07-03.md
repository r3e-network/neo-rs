## Architecture Audit Implementation - RPC Plugin Decoupling & neo-wallets Refactoring

### Completed Work

#### Task #143: Complete RPC Plugin Decoupling from neo_system::Node

**Problem:** `neo-rpc` plugin modules (application_logs, tokens_tracker) stored `Arc<neo_system::Node>` and used `Node` for handler downcasts, creating an unnecessary dependency on the composition root.

**Solution:** Replaced `Arc<Node>` with `Arc<ProtocolSettings>` in all plugin struct fields, and changed `CommittingHandler`/`CommittedHandler` downcasts from `Node` to `ProtocolSettings` across the workspace.

**Key changes:**
- `TrackerBase.neo_system: Arc<Node>` → `settings: Arc<ProtocolSettings>`
- `Tracker::on_persist`: removed unused `system: &Node` parameter
- Handler downcasts changed: `system.downcast_ref::<Node>()` → `system.downcast_ref::<ProtocolSettings>()`
- Callers in `neo-node/src/node/context/mod.rs` pass `self.settings.as_ref()` instead of `node.as_ref()`
- Updated `neo-oracle-service/src/service/handlers.rs` handler downcasts
- Removed dead `plugin_node()` method from `DaemonContext`
- Updated architecture test for default-members change

#### Task #141: Refactor neo-wallets to Remove Execution Dependency

**Problem:** `neo-wallets` (L6) depended on `neo-execution` (L4) for `Helper::signature_redeem_script()` and the `Contract` type, creating a layering issue.

**Phase 1 — Remove `Helper` dependency:**
- Replaced `Helper::signature_redeem_script(pk)` with `RedeemScript::signature_redeem_script(pk)` in `key_pair.rs`
- Removed `pub use neo_execution::Helper;` re-export from `lib.rs` (confirmed unused externally)

**Phase 2 — Move `Contract` to `neo-vm`:**
- Created `neo-vm/src/contract.rs` with the `Contract` struct
- Changed `get_address()` to take `address_version: u8` parameter (removed `neo-config` dependency)
- `neo-execution/src/contracts/contract.rs` now re-exports from `neo-vm` (backward compat)
- Updated `neo-wallets` imports: `use neo_execution::Contract` → `use neo_vm::Contract`
- Added `tracing` dependency to `neo-vm/Cargo.toml`

**Remaining:** `asset_descriptor.rs` still uses `ApplicationEngine` from `neo-execution` for smart contract calls.
